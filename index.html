Side projects

I love text based modeling software, like MscGen (sequence charts) and Graphviz (networks). Weirdly, tooling to work with that software is either non-existant or klunky. I'm on a mission to change that.

In 2013 I implemented mscgen in JavaScript and put it on a website (mscgen.js.org). So I could use it from anywhere. And so I could learn how others use it. That worked. Three years later there's a cli, a library on npm, two little languages that expand upon the ideas in MscGen, syntax highlighting in codemirror and visual studio code. And a plugin for the Atom editor. Which is really awesome. You should try it. No, really.

At the end of 2016 I set upon writing dependency-cruiser, a dependency analyzer and visualizer for TypeScript 3) to keep both my team and the chief architect happy. My team wanted to keep a mono-repo for a while to keep initial development manageable. Our chief architect was afraid we'd grow weird dependencies within that repo. With dependency-cruiser we could prove that concern unfounded, both visually and in our CI.

I have some other side projects; GitHub is doing a decent job on showcasing them, so if you want to see them, hit over there.
